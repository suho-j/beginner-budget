# 테스트 계획

## 자동 테스트

실행 위치: `beginner-budget-app/`

```bash
node --check js/storage.js && node --check js/transactions.js && node --check js/ui.js && node --check js/app.js
node tests/run-tests.cjs
```

자동 테스트 범위:

- 빈 저장소 기본값
- 손상된 JSON 복구
- localStorage 저장 실패 시 예외 없이 실패 결과 반환
- 실제 존재하지 않는 날짜 거부
- 로컬 날짜/월 포맷
- 손상된 거래 데이터 정규화
- 수입/지출/잔액/예산 계산
- 샘플 데이터 교체
- JSON 내보내기/가져오기

## 브라우저 수동 테스트

- 페이지 로딩 후 콘솔 오류가 없어야 한다.
- 수입 1건과 지출 1건을 추가하면 요약이 즉시 갱신되어야 한다.
- 새로고침 후 데이터가 유지되어야 한다.
- 월별 필터와 유형 필터가 동작해야 한다.
- 삭제 버튼은 확인 창 이후에만 삭제해야 한다.
- 샘플 데이터 버튼을 두 번 누르면 기존 샘플 교체 확인이 떠야 한다.
- JSON 내보내기/가져오기
- 빈 JSON 또는 잘못된 스키마 가져오기는 현재 데이터를 교체하지 않아야 한다.
- 잘못된 거래가 섞인 JSON은 유효 거래만 가져오고 제외 건수를 안내해야 한다.
- 1MB 초과 JSON 파일은 가져오기를 거부해야 한다.
- 중복 ID가 있는 JSON은 중복 ID를 재발급해야 한다.
- 전체 초기화는 확인 후 기본 상태로 돌아가야 한다.
- 금액/예산에 빈 값, 0, 음수, 소수를 넣으면 오류 메시지와 포커스가 표시되어야 한다.

## 반응형/접근성 테스트

- 360px 너비에서 가로 스크롤이 없어야 한다.
- 모든 입력은 라벨이 연결되어 있어야 한다.
- 오류 필드는 `aria-invalid="true"`로 표시되어야 한다.
- 키보드 Tab 순서로 입력과 버튼을 사용할 수 있어야 한다.
- 수입/지출/예산 상태는 색상뿐 아니라 텍스트로도 구분되어야 한다.
