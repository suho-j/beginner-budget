# 설계 문서

## 사용자 경험 방향

- 첫 화면에서 “들어온 돈, 쓴 돈, 남은 돈, 예산 남은 금액”을 카드로 보여준다.
- 거래 입력은 한 화면 안에서 끝나도록 필드를 최소화한다.
- 작은 화면에서는 표 대신 카드형 목록을 사용한다.
- 오류와 성공 상태는 한국어 문장으로 안내한다.

## 화면 구성

1. 헤더
   - 앱 이름: 처음 가계부
   - 초보자를 위한 한 줄 설명
2. 요약 카드
   - 수입 합계
   - 지출 합계
   - 잔액
   - 예산 남은 금액
3. 월 예산 설정
4. 수입/지출 입력 폼
5. 필터 및 데이터 도구
   - 월별 보기
   - 유형 보기
   - 샘플 데이터 채우기
   - 전체 초기화
6. 거래 내역 목록 및 빈 상태

## 접근성

- 모든 입력 요소에 명시적인 `label`을 연결했다.
- 삭제 버튼에는 거래 정보를 포함한 `aria-label`을 제공한다.
- 요약과 목록 영역은 변경 내용을 보조기술이 인지할 수 있도록 `aria-live`를 사용한다.
- 수입/지출/예산 초과는 색상과 함께 “수입 +”, “지출 -”, “예산 초과” 텍스트로 표시한다.

## 반응형 설계

- 기본은 1열 모바일 레이아웃이다.
- 560px 이상에서 요약 카드와 폼 일부를 2열로 배치한다.
- 900px 이상에서 데스크톱에 맞게 넓은 레이아웃을 사용한다.
