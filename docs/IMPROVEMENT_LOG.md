# 개선 진단 및 애자일 반복 로그

## 현재 현황

- 앱 위치: `beginner-budget-app/`
- 형태: 빌드 없는 정적 HTML/CSS/JavaScript 앱
- 구현 완료 기능: 거래 추가/삭제, 월별/유형 필터, 월 예산, 샘플 데이터, 전체 초기화, localStorage 저장, 문서/체크리스트
- 이전 검증: JS 문법 검사, HTML 파서 검사, 정적 서버 응답 확인, 기본 계산 로직 확인

## 개선 필요사항 백로그

### P0 안정성

- localStorage `setItem/removeItem` 예외 처리 추가
  - 원인: 저장소 차단, 비공개 모드, 용량 초과 시 사용자 액션이 런타임 오류로 중단될 수 있음
  - 개선: 저장/삭제 결과를 명시적으로 반환하고 UI에서 복구 메시지 표시

- 기본 날짜/월을 UTC가 아닌 사용자 로컬 날짜 기준으로 계산
  - 원인: `toISOString()`은 UTC 기준이라 KST 자정 근처에서 전날로 표시될 수 있음
  - 개선: `getFullYear/getMonth/getDate` 기반 포맷 함수 사용

- 저장소 복구 데이터 검증 강화
  - 원인: 손상된 localStorage에 잘못된 날짜/카테고리가 들어와도 일부 살아남을 수 있음
  - 개선: 실제 달력 날짜, 유형별 카테고리, 메모 길이 검증

### P1 UX/접근성

- 커스텀 검증 메시지를 입력 필드와 연결
  - 개선: `aria-invalid`, `aria-describedby`, 첫 오류 필드 focus

- 샘플 데이터 중복 추가 방지
  - 개선: 같은 월에 샘플 데이터가 있으면 확인 후 교체/중단

- 예산 입력 중 필터 변경 시 입력값이 덮어써지는 문제 방지
  - 개선: 렌더 시 focus된 예산 입력은 덮어쓰지 않음

- 금액 입력 도움말 제공
  - 개선: 쉼표 없이 숫자만 입력하라는 힌트와 검증 메시지 제공

### P2 SaaS 품질 보강

- 예산 사용률 진행바와 상태 배지 추가
- 데이터 내보내기/가져오기(JSON) 추가로 localStorage 삭제 리스크 완화
- 자동 테스트 파일 추가
- `IMPLEMENTATION_PLAN.md` 한국어화
- 브라우저 렌더링/콘솔 오류 검증

## 반복 계획

1. P0 안정성 수정
2. P1 UX/접근성 수정
3. P2 품질 보강 중 정적 앱에 맞는 범위 구현
4. 자동 테스트와 브라우저 검증

## 완료된 반복

### 1차 개선

- localStorage 저장/삭제 실패를 try/catch로 처리하고 UI에 실패 메시지를 표시했다.
- 기본 날짜와 월을 UTC가 아닌 로컬 날짜 기준으로 계산하도록 수정했다.
- 실제 존재하지 않는 날짜, 잘못된 유형별 카테고리, 잘못된 금액을 정규화 단계에서 제외했다.
- 예산 사용률 진행바, JSON 내보내기/가져오기, 샘플 데이터 교체 흐름을 추가했다.
- 필드 오류 메시지와 `aria-invalid` 처리를 추가했다.

### 2차 개선

- 독립 리뷰에서 발견된 JSON 가져오기 데이터 손실 리스크를 개선했다.
- 빈 JSON, 잘못된 스키마, 유효 거래 0건 가져오기를 차단했다.
- 잘못된 거래 제외 건수를 가져오기 확인 메시지에 표시했다.
- 중복 거래 ID를 정규화 단계에서 재발급하도록 했다.
- 가져오기 파일 크기를 1MB 이하로 제한했다.
- progressbar에 `aria-valuetext`를 추가했다.
- 삭제 버튼 터치 높이를 44px로 맞췄다.

### 3차 개선

- 최종 리뷰에서 지적된 import 오류 경로의 접근성 누락을 보완했다.
- 파일 가져오기 실패 시 숨겨진 file input에 `aria-invalid="true"`를 설정하고 가져오기 버튼으로 포커스를 복귀시킨다.
